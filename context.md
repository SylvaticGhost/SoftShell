Розробити консольний застосунок **SoftShell** (CLI) з REPL-інтерфейсом (Read-Eval-Print Loop), який виступатиме в ролі "Bash-to-PowerShell транслятора" та інтелектуального помічника.

# Ключові вимоги:

Підтримка чотирьох типів вводу: прямі команди PowerShell, Bash-команди для перекладу (через /t), текстові запити природною мовою (через /a) та голосові запити (через /av).

Запобігання автоматичному виконанню згенерованого коду: система має показати команду, її опис і запропонувати меню дій — [E]xecute, [C]opy, [R]eject.

Підтримка контексту поточної директорії (CWD) між виконанням команд.

# Вибір технологій

.NET 10. CentralPackageManagement

Інтерфейс (TUI): Spectre.Console для красивого REPL, форматування виводу та інтерактивних меню.

Голосове введення: Нативна бібліотека System.Speech для розпізнавання голосу безпосередньо у Windows без додаткових залежностей.

Інтеграція ШІ: OpenAI SDK для підключення до Groq API (модель llama3 або аналогічна), що забезпечить блискавичну генерацію відповідей.

Буфер обміну: Бібліотека TextCopy для кросплатформної роботи з clipboard.

# Етапи виконання
### Створення інтерфейсу
   Реалізація головного REPL-циклу, який очікує ввід користувача (імітація сесії терміналу).

Розробка класифікатора вводу (Recognizer), який розбирає рядок на наявність префіксів (/t, /a, /av) або ідентифікує прямі команди.

Створення інтерактивного віджету через Spectre.Console для підтвердження дій користувачем (Виконати/Скопіювати/Відхилити).

### Виконання команд PowerShell
Налаштування виклику powershell.exe у підпроцесі (System.Diagnostics.Process) з перехопленням stdout та stderr.

Реалізація архітектурного хаку з "маркером директорії" (додавання (Get-Location).Path в кінець кожної команди) для відслідковування команди cd і збереження поточного контексту терміналу між сесіями.

### Словник та довідник (підказки)
Створення гібридної системи обробки (Rule-based Fast Path) для економії токенів і зменшення затримок.

Розробка статичного bash_to_ps_dictionary для миттєвого мапінгу найпопулярніших команд (наприклад, ls -> Get-ChildItem).

Створення локального ps_docs_dictionary (або інтеграція з Get-Help) для підтягування описів команд без звернення до інтернету.

### Інтеграція AI
Налаштування клієнта Groq API через OpenAI SDK.

Створення спеціалізованих системних промптів для двох сценаріїв:

Переклад складних Bash-команд з прапорцями на ідіоматичний PowerShell.

Генерація PowerShell-команд на основі текстових або голосових запитів природною мовою.

Налаштування моделі на повернення строгого JSON-формату (Command, Explanation) для зручної десеріалізації в об'єкти C# та подальшого виводу в інтерфейс.
